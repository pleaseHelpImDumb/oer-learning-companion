Introduction

Welcome to the Achievo Academy API documentation. This API powers the Achievo learning platform, allowing users to manage study sessions, track progress, interact with an AI tutor, and earn tokens.

Authentication

Most endpoints require authentication via a JWT stored in an HTTP-only cookie (jwt). Requests must be made from an allowed origin (CORS).

Authentication

POST /users/register
Public

Registers a new user account.

Request Body 

{
  "username": "johndoe",
  "email": "john@example.com",
  "password": "securepassword123"
}

Response (201 Created) 

{
  "user": { "userId": 1, "username": "johndoe", "email": "john@example.com" },
  "csrfToken": "random-csrf-string",
  "message": "Welcome, johndoe! Your account has been registered."
}
POST /users/login
Public

Authenticates a user and sets a JWT cookie.

Request Body 

{
  "identity": "johndoe", // username or email
  "password": "securepassword123"
}

Response (200 OK) 

{
  "message": "Log-in success!",
  "user": { "userId": 1, "email": "...", "name": "..." },
  "csrfToken": "random-csrf-string"
}
POST /users/logout
Public

Clears the JWT cookie.

Response (200 OK) 

{ "message": "Logged out" }
POST /users/forgot-password

Initiates password reset process. Sends an email with a reset token if the email exists.

Body 

{ "email": "john@example.com" }
POST /users/reset-password

Resets the password using the token received via email.

Body 

{ "token": "...", "newPassword": "..." }

User Profile

GET /users/profile
Auth Required

Retrieves the current authenticated user's profile information, including stats and badges.

Response (200 OK) 

{
  "user": {
    "userId": 1,
    "username": "johndoe",
    "email": "john@example.com",
    "role": "USER",
    "avatarUrl": "https://...",
    "totalTokensEarned": 50,
    "track": { "id": 1, "name": "CS", "description": "..." },
    "badges": [...]
  }
}
POST /users/onboard
Auth Required

Completes the user profile setup (onboarding). Sets preferences like study track, nickname, etc.

Request Body 

{
  "favoriteQuote": "Stay hungry, stay foolish.",
  "avatarUrl": "https://example.com/avatar.png",
  "checkInIntervalMinutes": 30,
  "trackId": 1,
  "nickname": "Johnny",
  "yearLevel": 2,
  "major": "Computer Science"
}
GET /users/stats

Returns lifetime study stats.

GET /users/week-stats

Returns study statistics for the past 7 days.

POST /users/add-break

Increments the user's break count.

POST /users/consume-token-24hrs

Spends tokens earned in the last 24 hours.

Sessions

Endpoints to manage study sessions. All routes require authentication.

Session Lifecycle
POST /sessions/start

Starts a new study session. Body: {"sessionGoalMinutes": 60}

GET /sessions/active

Gets the current active session and calculates live study time.

POST /sessions/:id/pause

Pauses the session.

POST /sessions/:id/resume

Resumes a paused session.

POST /sessions/:id/complete

Completes session, calculates rewards, updates stats.

POST /sessions/:id/cancel

Cancels session (no rewards).

POST /sessions/:id/notes

Sets notes for a specific session.

GET /sessions/:id/notes

Retrieves notes for a session.

POST /sessions/:id/consume-token

Spends tokens during an active session.

POST /sessions/:id/break

Creates a wellness check entry.

AI Chat

POST /ai/chat
Auth Required

Interacts with the AI tutor. Requires an active study session.

Request Body 

{
  "message": "Can you explain the concept of recursion?",
  "supportLevel": 2 // 1 = Hint, 2 = Guided, 3 = Full Scaffolding
}

Response (200 OK) 

{ "response": "Certainly! Recursion is..." }
GET /ai/history
Auth Required

Retrieves the chat history for the current active session.

Error Handling

Code Description
400 Bad Request – Invalid input or validation error.
401 Unauthorized – Missing or invalid JWT.
403 Forbidden – Valid token, but access denied (e.g., no active session).
404 Not Found – Resource does not exist.
409 Conflict – Resource conflict (e.g., user already exists, active session already running).
500 Internal Server Error – Something went wrong on the server.
